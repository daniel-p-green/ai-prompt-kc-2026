# Work Logs

This folder records project thinking in chronological order.

Purpose:

- show how the concept evolved, not just the polished answer
- preserve tradeoffs, dead ends, and decisions with timestamps
- give judges or reviewers evidence of iteration and AI-directed reasoning

Format:

- one dated file per work session
- reverse-chronological entries inside each file are acceptable, but simple timeline order is preferred
- each entry should include:
  - time
  - working question
  - decision or current belief
  - evidence or rationale
  - next move

Rule:

- keep the log honest
- record uncertainty when it exists
- do not rewrite early entries to sound smarter after the fact
