## Goal

Deliver one coherent StreetCart KC package that supports Oracle, Muse, and a thin Architect slice without splitting the team across three unrelated ideas.

## Inputs

- `docs/discovery/idea-brief.md`
- `docs/product/prd.md`
- `docs/product/jtbd.md`
- `docs/product/use-cases.md`
- `docs/engineering/test-strategy.md`

## Success Contract

- Done means: the team can show one sentence, one pilot, one budget story, one brand system, and one lightweight product demo that all describe the same company.
- Tested by: challenge-data-backed ZIP choices, internally consistent pilot math, a clean brand narrative, and a clickable or screen-based demo path for judges.
- No changes to: full delivery logistics, partner ERP replacement, or broad case-management workflows.

## Execution Plan (3 Steps)

1. Lock Oracle first: thesis, 60-day pilot, budget framing, ZIP targets, KPIs, and objections.
2. Build Muse second: StreetCart KC brand, tournament framing, scoreboard mock, wrap concept, and resident-facing collateral.
3. Build Architect last and thin: one dashboard or flow that proves the system can schedule hubs, notify households, and update the leaderboard.

## Task Breakdown

### Task 1

- Files to modify: `docs/product/prd.md`, `docs/discovery/idea-brief.md`, `docs/product/streetcart-kc-spine.md`
- Tests to add first: challenge the thesis against feasibility, prove which ZIPs are in scope, and lock the budget interpretation of `$500k revenue / $150k direct opex`
- Definition of done: Oracle materials explain the pilot in one read and answer the obvious judge objections

### Task 2

- Files to modify: `docs/product/streetcart-kc-spine.md`, brand assets, pitch visuals, scoreboard mock, wrap mock, flyer copy
- Tests to add first: every visual still shows the food access metric, target ZIPs, and 60-day pilot instead of drifting into a pure marketing stunt
- Definition of done: Muse assets make the concept feel distinctly KC while staying tied to operations and outcomes

### Task 3

- Files to modify: thin demo files, Figma, or app scaffold for a coordinator dashboard plus public leaderboard and resident instructions
- Tests to add first: the demo can show hub creation, resident notification, and leaderboard update in under two minutes
- Definition of done: Architect exists as proof, not as a time sink

## Risk Notes

- If the pitch over-indexes on sports and wraps, judges may think the concept is gimmicky.
- If you try to fully build Architect before Oracle and Muse are locked, the whole package will get thinner.
- If the pilot assumes direct streetcar cargo operations in 60 days, feasibility will get attacked.
